# Tier-3.2: Split `playlist_generator.py` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce `src/playlist_generator.py` from 4,038 LOC to ~1,400 LOC by extracting the six largest orchestrator methods (~2,600 LOC) into a new `src/playlist/orchestrator/` package, with `PlaylistGenerator` becoming a thin façade that delegates.

**Architecture:** Follow the proven Tier-3.1 pattern — verbatim moves of method bodies into free functions that take explicit dependencies as parameters; class methods become thin wrappers that pass `self.config`, `self.library`, `self.metadata`, `self.lastfm`, etc. into the free functions. New code lives in `src/playlist/orchestrator/*.py`. The class API is preserved exactly — no caller changes outside `playlist_generator.py`.

**Tech Stack:** Python 3.11+, ruff/mypy/pytest. No new dependencies. Pattern mirrors existing `src/playlist/history_analyzer.py` delegation style.

---

## Why this works

The existing phase 2–9 submodules (`utils`, `filtering`, `scoring`, `diversity`, `ordering`, `candidate_generator`, `reporter`, `history_analyzer`) already established the delegation pattern. ~37 method bodies in `playlist_generator.py` already delegate. The remaining mass (~2,600 LOC) lives in six fat orchestrator methods that haven't been touched. This plan extracts them using the same pattern.

## Dependency order

`_maybe_generate_ds_playlist` is called by **all four** public orchestrators. It must be extracted **before** the orchestrators that depend on it, otherwise each orchestrator extraction has to thread the `_maybe_generate_ds_playlist` body through.

Dependency graph (caller → callee):
- `create_playlist_for_artist` → `_maybe_generate_ds_playlist` + many helpers
- `create_playlist_for_genre` → `_maybe_generate_ds_playlist`
- `create_playlist_from_seed_tracks` → `_maybe_generate_ds_playlist`
- `create_playlist_batch` → `create_playlist_for_artist` + `_create_playlists_from_single_artists`
- `_create_playlists_from_single_artists` → `create_playlist_for_artist`

→ extract `_maybe_generate_ds_playlist` first, then independent leaves, then artist, then batch.

from tests.conftest_playlist_generator import synthetic_generator  # noqa: F401

GOLDENS_DIR = Path(__file__).parent / "goldens" / "playlist_generator"

def _snapshot_result(tracks):
    """Stable subset of fields for golden comparison."""
    return [{"id": t.get("rating_key"), "artist": t.get("artist"), "title": t.get("title")} for t in tracks]

    """Snapshot the orchestrator output; regenerate with --generate-golden."""

        tracks = synthetic_generator.create_playlist_for_artist(
            artist_name="Artist 0", track_count=5, use_artist_style=False,
        )
    elif scenario == "genre_basic":
        tracks = synthetic_generator.create_playlist_for_genre(
            genre="genre_0", track_count=5,
        )
    elif scenario == "seed_list_basic":
        tracks = synthetic_generator.create_playlist_from_seed_tracks(
            seed_track_ids=["t0", "t4"], track_count=5,
        )
    snapshot = _snapshot_result(tracks)
    golden_path = GOLDENS_DIR / f"{scenario}.json"
    if request.config.getoption("--generate-golden", default=False):
        golden_path.parent.mkdir(parents=True, exist_ok=True)
        golden_path.write_text(json.dumps(snapshot, indent=2, sort_keys=True) + "\n")
        return
    expected = json.loads(golden_path.read_text())
    assert snapshot == expected, f"Golden mismatch for {scenario}"

## Risk register

| Risk | Mitigation |
|------|------------|
| Hidden `self.*` access missed during cataloging → `AttributeError` at runtime | Smoke goldens (PR-0) catch behavioral changes. Ruff catches the obvious undefined-name cases. |
| Bound-method callbacks differ subtly from free functions (default args, decorators) | Use bound methods directly when delegating (`self._method`). Don't try to be clever. |
| `__init__.py` re-export creates a circular import | Defer the re-export to PR-7. Until then, callers use the full path `from src.playlist.orchestrator.ds import maybe_generate_ds_playlist`. |
| Smoke goldens too narrow → regressions slip through | Run the full `pytest -q -m "not slow"` suite after each PR, not just the goldens. |
| `_artist_style.py` and the artist orchestrator are co-evolving | Watch `tests/test_artist_style.py` carefully during PR-5. If it fails, the artist-style setup logic has been touched and needs re-verification against `git show HEAD~1:src/playlist_generator.py`. |
| User has uncommitted WIP (`max_non_seed_tracks_per_artist`) → conflicts | The WIP is in `pipeline/core.py` and tests, not `playlist_generator.py`. Should not conflict, but check `git status` before each PR. |
