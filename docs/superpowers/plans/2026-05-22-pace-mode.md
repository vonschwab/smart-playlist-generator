# Pace Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Decouple rhythm from the combined sonic axis. Treat timbre+harmony ("color") as one perceptual dimension and rhythm/pace as a separate, independently-tunable dimension. Expose as a `pace_mode` slider (`strict`/`narrow`/`dynamic`) alongside the existing sonic and genre mode sliders. Default `dynamic` preserves current behavior exactly.

**Architecture:** `X_sonic` is stored as `[sqrt(w_r)·rhythm_pca | sqrt(w_t)·timbre_pca | sqrt(w_h)·harmony_pca]`. Slicing into per-tower sub-vectors and computing cosine gives pure per-axis similarity — the `sqrt(w)` scale factor is normalized away by cosine. Pace is enforced at two layers:

1. **Admission floor (Tier 1):** Each candidate must clear a rhythm-similarity floor against the seeds (max-over-seeds). Mode-driven threshold.
2. **Per-segment moving target (Tier 2):** Inside the beam, at step `s` of segment length `L`, the candidate's rhythm must hug a target rhythm `target_r(s) = (1 − s/L)·R_A + (s/L)·R_B`. Tolerance is mode-driven. Naturally collapses to "stay anchored" when piers share rhythm, and "interpolate arc" when they differ.

Transition weights and beam edge scoring are unchanged — pace is enforced as a gate, not a transition penalty, so beam-reporter alignment is untouched.

**Tech Stack:** Python 3.11, numpy, PySide6, pytest

**Out of scope:**
- Auto-detecting per-segment whether to anchor vs. arc (the moving-target formulation unifies both)
- Adding rhythm penalty to beam edge scoring (it's a gate, not a soft term)
- Per-step rhythm target adjustments based on micro-piers, detours, etc.
- Backward compatibility for any renamed config (user is in extended testing, sole operator)

---

## Mode parameters (initial values — tunable)

| Mode | `admission_floor` | `bridge_floor` | Description |
|---|---|---|---|
| `strict` | 0.55 | 0.65 | Tight tempo fidelity; stay slow/fast/whatever the seeds are |
| `narrow` | 0.35 | 0.45 | Moderate tempo anchoring |
| `dynamic` | 0.00 | 0.00 | No pace constraint (current behavior) |

`admission_floor`: candidate's `max(cosine(R_c, R_seed_i)) >= admission_floor`. Per-candidate gate at global pool construction.

`bridge_floor`: at beam step `s` of segment length `L`, with target `T(s) = (1−s/L)·R_A + (s/L)·R_B`, require `cosine(R_c, T(s)) >= bridge_floor`. Per-candidate gate at beam expansion.

If too few candidates pass `bridge_floor` for a step, the existing infeasibility backoff (`infeasible_handling.backoff_steps`) is reused — pace floor relaxes alongside `bridge_floor` in lockstep.

```python
# src/playlist/sonic_axes.py
"""Slice the combined sonic PCA vector into per-tower sub-vectors.

X_sonic is stored as a concatenation:
    [sqrt(w_r) * rhythm_pca | sqrt(w_t) * timbre_pca | sqrt(w_h) * harmony_pca]

Cosine similarity normalizes magnitude, so slicing and computing cosine on
each sub-block recovers pure per-axis similarity — the sqrt(w) scaling does
not affect the result.
"""
from __future__ import annotations

from typing import Dict, Tuple

import numpy as np


def extract_axis_vectors(
    X_sonic: np.ndarray,
    *,
    tower_pca_dims: Tuple[int, int, int],
) -> Dict[str, np.ndarray]:
    """Return per-axis views into X_sonic.

    Args:
        X_sonic: (N, D) where D == sum(tower_pca_dims).
        tower_pca_dims: (rhythm_dim, timbre_dim, harmony_dim).

    Returns:
        Dict with keys "rhythm", "timbre", "harmony", "color".
        "color" is the concatenation of timbre and harmony slices.
    """
    r_dim, t_dim, h_dim = tower_pca_dims
    expected = r_dim + t_dim + h_dim
    if X_sonic.shape[1] != expected:
        raise ValueError(
            f"X_sonic has {X_sonic.shape[1]} dims, but sum of tower_pca_dims "
            f"is {expected} (rhythm={r_dim}, timbre={t_dim}, harmony={h_dim})"
        )
    rhythm = X_sonic[:, :r_dim]
    timbre = X_sonic[:, r_dim : r_dim + t_dim]
    harmony = X_sonic[:, r_dim + t_dim : r_dim + t_dim + h_dim]
    color = X_sonic[:, r_dim:]  # timbre + harmony
    return {
        "rhythm": rhythm,
        "timbre": timbre,
        "harmony": harmony,
        "color": color,
    }


def axis_cosine_similarity(a: np.ndarray, b: np.ndarray) -> np.ndarray:
    """Row-wise cosine similarity. a: (N, D), b: (M, D) → (N, M)."""
    a = np.asarray(a, dtype=float)
    b = np.asarray(b, dtype=float)
    if a.ndim == 1:
        a = a.reshape(1, -1)
    if b.ndim == 1:
        b = b.reshape(1, -1)
    a_norm = a / (np.linalg.norm(a, axis=1, keepdims=True) + 1e-12)
    b_norm = b / (np.linalg.norm(b, axis=1, keepdims=True) + 1e-12)
    return a_norm @ b_norm.T


def interpolate_axis_vector(
    R_a: np.ndarray,
    R_b: np.ndarray,
    t: float,
) -> np.ndarray:
    """Linear interpolation between two axis vectors. t in [0, 1]."""
    return (1.0 - float(t)) * np.asarray(R_a, dtype=float) + float(t) * np.asarray(R_b, dtype=float)
```

```python
# ============================================================================
# PACE (RHYTHM/TEMPO) MODE PRESETS
# ============================================================================

PACE_MODE_PRESETS: Dict[str, Dict[str, Any]] = {
    "strict": {
        "admission_floor": 0.55,
        "bridge_floor": 0.65,
        "description": "Tight tempo fidelity - stay anchored to seed pace",
        "use_case": "Slow/meditative seeds; mood-locked playlists",
    },
    "narrow": {
        "admission_floor": 0.35,
        "bridge_floor": 0.45,
        "description": "Moderate tempo anchoring",
        "use_case": "Consistent energy with some flex",
    },
    "dynamic": {
        "admission_floor": 0.00,
        "bridge_floor": 0.00,
        "description": "No pace constraint (default)",
        "use_case": "Multi-tempo playlists; current behavior",
    },
}


def resolve_pace_mode(mode: str, overrides: Optional[Dict[str, Any]] = None) -> Dict[str, Any]:
    """Resolve pace mode string to settings dict.

    Pace controls rhythm-axis admission and per-segment moving-target gates.
    Default is "dynamic" (no pace constraint).
    """
    mode = (mode or "dynamic").lower()
    if mode not in PACE_MODE_PRESETS:
        valid = ", ".join(PACE_MODE_PRESETS.keys())
        raise ValueError(f"Unknown pace mode: '{mode}'. Valid modes: {valid}")
    settings = {k: v for k, v in PACE_MODE_PRESETS[mode].items() if k not in {"description", "use_case"}}
    if overrides:
        settings.update(overrides)
    return settings
```

```python
# src/playlist/pier_bridge/pace_gate.py
"""Per-step rhythm-target gate for pier-bridge beam search.

At step s (1-indexed, s in [0, L]) of a segment of interior length L,
the target rhythm vector interpolates linearly between pier_A and pier_B:

    target(s) = (1 - s/L) * R_A + (s/L) * R_B

When piers share rhythm (R_A ≈ R_B), this collapses to anchored behavior.
When they differ, this produces a smooth tempo arc through the segment.
"""
from __future__ import annotations

from typing import List, Sequence

import numpy as np

from src.playlist.sonic_axes import axis_cosine_similarity, interpolate_axis_vector


def compute_step_rhythm_target(
    R_a: np.ndarray,
    R_b: np.ndarray,
    *,
    step: int,
    segment_length: int,
) -> np.ndarray:
    """Linear interpolation between pier rhythm vectors.

    Args:
        R_a: pier_A rhythm sub-vector.
        R_b: pier_B rhythm sub-vector.
        step: 0-indexed position within the segment (0 = at pier_A, L = at pier_B).
        segment_length: number of *interior* steps. Step L lands at pier_B.

    Returns:
        Target rhythm vector at this step.
    """
    if segment_length <= 0:
        return np.asarray(R_a, dtype=float)
    t = max(0.0, min(1.0, float(step) / float(segment_length)))
    return interpolate_axis_vector(R_a, R_b, t)


def filter_candidates_by_rhythm_target(
    *,
    candidate_indices: Sequence[int],
    rhythm_matrix: np.ndarray,
    target: np.ndarray,
    floor: float,
) -> List[int]:
    """Return indices of candidates whose rhythm cosine to target >= floor."""
    if floor <= 0.0:
        return list(candidate_indices)
    candidate_indices = list(candidate_indices)
    if not candidate_indices:
        return []
    R = rhythm_matrix[candidate_indices]  # (k, dim)
    sim = axis_cosine_similarity(R, target.reshape(1, -1)).reshape(-1)
    return [idx for idx, s in zip(candidate_indices, sim) if float(s) >= float(floor)]
```
