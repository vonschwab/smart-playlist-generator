# GUI audit — mobile functionality + visual discipline (2026-07-09)

Deep audit of the web GUI against `docs/UI_UX_DISCIPLINE.md` (written together with this
report). Method: two code-audit passes (mobile functionality; visual/design consistency), a
UX-principles research pass (Apple HIG / WCAG 2.2 / WebKit / NN/g), and a live evidence sweep —
`web/tests/mobile-audit.spec.ts` run at 390×844 with touch emulation on **both Chromium and
WebKit** against the built app + fake worker.

**Evidence:** `web/test-results/mobile-audit/` and `mobile-audit-webkit/` (7 screenshots each +
`measurements.json` with rendered touch-target and font sizes). WebKit rendered
near-identically to Chromium — no Safari-engine layout breakage; the iOS risks below are
runtime behaviors, found by code audit.

**Headline:** the mobile shell's architecture is genuinely sound — clean 1024px shell swap,
every feature reachable on the phone, zero horizontal overflow on all 7 surfaces, native
selects, reduced-motion + focus-visible floors in place. The failures are one severe lifecycle
bug, a systematic touch-target/typography deficit, and design-system erosion (two parallel
color systems, four button recipes).

## P1 — Degraded on the phone

- **`h-screen` root** (100vh): bottom tab bar sits under Safari's dynamic toolbar. Fix `h-dvh`. — `Shell.tsx:24`
- **Every input/select in the app < 16px** → iOS zooms on each focus (seed input `text-[11px]`, tracks count, dialog/name/search inputs at `text-xs`/`text-[10px]`). The `pointer:coarse` floor in `index.css:46-51` bumps display text only to 11–13px and skips form controls entirely. — `GenerateControls.tsx:266,315`, `BlacklistPanel.tsx:105`, `EditGenresDialog.tsx:105`, `ExportM3U8Dialog.tsx:44`, more
- **Touch targets fail wholesale** (live-measured): main Generate button 103×28; all toolbar controls 26–30px tall; play/kebab **12×20**; Tools' 19 checkboxes **13×13**; dialog close **8×18**; tab bar 40px (44 needed). Only the WCAG 24px floor + spacing exception makes the table rows defensible — the rest need height. — `measurements.json`
- **Controls own ~58% of the phone viewport even after generation**; the playlist lives in the bottom third (screenshots `01`/`03`). Row 1 of GenerateControls never collapses on mobile. — `GenerateControls.tsx:239-414`
- **Dialogs are viewport-centered with no `max-h`/scroll and no keyboard awareness** — a focused input can end up behind the iOS keyboard. — all four dialogs, e.g. `ExportM3U8Dialog.tsx:32`
- **Artist-autocomplete dropdown isn't portaled** out of the root's `overflow-hidden` (StylePopover was fixed for exactly this); likelier to clip on mobile with the keyboard up. — `GenerateControls.tsx:268-292`
- **`text-[8px]` labels** escape the coarse-pointer floor (it only covers 9–11px); DiagnosticsPanel is largely 8px on the mobile-reachable Diag tab. — `DiagnosticsPanel.tsx:55-69`, `BlacklistPanel.tsx:81`
- **MiniPlayer transport** ⏮/⏭ are tiny glyph buttons; the seek bar is a 2px strip. — `MiniPlayer.tsx:51-72`

## P2 — Design-system erosion (desktop + mobile)

Scorecard from the visual audit: token adherence **3/10**, typography **3/10**, contrast
**5/10**, spacing **8/10**, component consistency **5/10**, states **6/10**, copy **5/10**,
hierarchy **6/10**, craft **6/10**.

- **Two parallel color systems:** 185 raw-hex occurrences in 16 files; the generate-flow core hand-copies token values (`GenerateControls.tsx:15-16`, `ToolsPanel.tsx`, `DiagnosticsPanel.tsx`, `BlacklistPanel.tsx`, `App.tsx:230-234`), newer review-flow files use theme classes correctly. A retint currently means editing 16 files.
- **Contrast failures** (computed): `faint #5b6470` on panel = **2.96:1 FAIL** — and it carries real copy (entire LogPanel stream, empty states, hints). Undocumented gray `#3a3f4b` = **1.68:1** on section headers. Everything else passes (muted 5.7:1, text 14.6:1, accent 12.8:1, chipText 8.8:1).
- **Type scale is de facto arbitrary:** 194 `text-[Npx]` (8px×6, 9px×26, 10px×123, 11px×39) vs 52 real-scale uses; the coarse-pointer `[class*="text-[9px]"]` substring hack patches over it and breaks on the first computed class string.
- **Ad-hoc severity palettes:** three components invented their own red/orange/purple/blue (`DiagnosticsPanel.tsx:4-6`, `BlacklistPanel.tsx:5`, `RelaxationNotice.tsx:19-30` with stock Tailwind `yellow-900`/`blue-300`) instead of `warn`/`danger` + a missing `info` token.
- **Four "primary button" recipes, three ghost recipes** (`ToolsPanel.tsx:61-74` vs dialog buttons vs review-panel accepts); de facto row-highlight token `#15202b` used in three files but never promoted.
- **States:** JobsPanel has **no empty state** (black void — screenshot `02-tab-jobs`); LogPanel likewise before first log; loading is designed once (ToolsPanel progress bar), plain text elsewhere.
- **Copy:** raw stage names as checkbox labels (`muq`, `genre-sim`, `artifacts` — `ToolsPanel.tsx:10-14,309`), literal `"Blacklist 1 Track(s)"` (`TrackContextMenu.tsx:46`), every error surfaced as unprocessed `Error: …` (`api.ts:22-26` + all catch sites).

## P3 — Polish

- `index.html`: Vite default title + favicon; no `theme-color`, `apple-touch-icon`, manifest, or `viewport-fit=cover`.
- Google Fonts CDN `@import` — render-blocking, third-party, in a local-first app (`index.css:1`).
- `type="number"` without `inputMode="numeric"` (`GenerateControls.tsx:309-316`).
- No `overscroll-behavior` on inner scroll panes; no `-webkit-tap-highlight-color` handling.

## What's already right (don't re-fix)

Single-breakpoint shell swap with full feature reachability; resizable panels correctly
desktop-only; zero horizontal overflow everywhere (both engines); native `<select>`s; global
`:focus-visible` + `prefers-reduced-motion`; review-panel keyboard shortcuts mirrored as
buttons; backend audio Range/206 support already iOS-correct; disciplined spacing rhythm;
consistent Radix dialog chrome; several excellent empty states (TrackTable, review panels).

## Fix roadmap (proposed)

1. **Lifecycle (P0.1)** — WS reconnect w/ backoff + `visibilitychange` reconcile; wire `useJobReconcile` into the Generate flow. *The one bug that breaks the core feature.*
2. **Mechanical mobile floor** — `h-dvh`; 16px coarse-pointer input floor; ≥24/44px target floor (kebab, play, checkboxes, dialog ×, tab bar); coarse-pointer visibility for `group-hover` reveals; popover clamp + dialog `max-h`; autocomplete portal.
3. **Token/type rationalization** — sweep 185 raw hex onto tokens; add `info` + row-highlight tokens; raise or demote `faint`; kill `#3a3f4b`; collapse `text-[Npx]` onto the scale (retiring the substring hack); extract Button/Chip components.
4. **Mobile layout intent** — collapse controls after generation (playlist owns the viewport), thumb-zone primary actions, designed empty states, `friendlyError()`, stage-label glosses, app identity (title/icon/theme-color/manifest), self-hosted fonts.

Each phase is independently shippable and verifiable via the sweep + discipline doc greps.
